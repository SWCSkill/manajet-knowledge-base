# 4.2 Статистики. Как создать

Публичная ссылка: https://app.manajet.org/FAQ/Public/Index/149?culture=ru-RU

Добавление статистик в ManaJet выполняется с помощью процедуры из нескольких шагов, где на каждом шаге нужно выбрать или настроить определенные свойства создаваемой статистики.
Добавлять статистики в программу ManaJet может сотрудник, которому назначены права на создание статистик (во вкладке “Сотрудники”).
Для создания статистики нужно войти в Личный кабинет (иконка ), выбрать вкладку “Статистики” и нажать кнопку “Добавить статистику”:
Заполняем первую страницу блока добавления статистики “Общие параметры”:
Здесь нужно указать:
Название
Полное название статистики. Например, “Число обработанных заказов”.
Ответственный
Из списка выбрать сотрудника, ответственного за данную статистику. Статистика без выбранного сотрудника не может быть создана.
Кто вводит
Указывается тот, кто будет вводить данные в эту статистику.
По умолчанию выбрано “Лично”. (То есть тот, кто выбран ответственным в предыдущем пункте). Но можно выбрать и другого сотрудника, который будет вводить данные в статистику, например, когда личный помощник вводит данные за руководителя. Также это может быть в ситуациях, когда не у всех сотрудников есть доступ к компьютеру. (Например, мастер участка вводит статистики рабочих и т.д.)
Единицы измерения
Указывается в чем измеряется данная статистика. Например, руб., шт, баллы и т.д.
Затем нажимаем кнопку “Далее”.
На странице “Данные” необходимо заполнить нужные вам поля:
Тип статистики
Из раскрывающегося списка выбрать нужный тип статистики. По умолчанию программа выбирает “Обычная”.
Типы статистик:
Обычная
Статистика, где сотрудник вручную вводит данные.
Групповая
Данная статистика включает в себя сумму нескольких статистик. Например, групповая статистика “Сумма продаж отдела” включает в себя статистики “Сумма продаж менеджера Иванова”, “Сумма продаж менеджера Петрова” и “Сумма продаж менеджера Васечкина”.
Формульная
Выполняет действия над вложенными статистиками, по формуле. С помощью формульной статистики можно, например, автоматически подсчитать сумму выручки компании, делённую на количество сотрудников. Или высчитать отношение суммы наличных к сумме счетов поставщиков. И т.д.
Мульти
Данный тип статистики позволяет на один экран вывести графики нескольких статистик, например, одновременно графики выручки, маржинального и скорректированного дохода. Или одновременно статистики нескольких сотрудников.
Внешняя
Данный тип статистики позволяет выгружать данные из внешних баз данных.
Выручка
Про данный тип статистики можно изучить в другой справочной статье https://app.manajet.org/FAQ/Public/Index/138?culture=ru-RU
Маржинальный доход
Это статистика, которая будет отображать разницу между выручкой и отчислениями в фонды выручки.
Скорректированный доход
Скорректированный доход - это разница между маржинальным доходом и отчислениями в фонды маржинального дохода. (Также можно создать фонды, куда откладываются средства со скорректированного дохода).
Расходы
Данная статистика будет автоматически заполняться,ссылаясь на данные в ФП.
Одобренные расходы
Данная статистика будет автоматически заполняться,ссылаясь на данные в ФП.
Новые карточки ЦРМ
Данная статистика будет автоматически заполняться,ссылаясь на данные в CRM. В данной статистике будет отображаться кол-во созданных карточек.
Счета поставщиков, количество
Данная статистика будет автоматически заполняться,ссылаясь на данные в Счетах поставщиков. Будет отображаться кол-во счетов.
Счета поставщиков, сумма
Данная статистика будет автоматически заполняться,ссылаясь на данные в Счетах поставщиков. Будет отображаться сумма счетов.
Счета клиентам, количество
Данная статистика будет автоматически заполняться,ссылаясь на данные в Счетах клиентам. Здесь будет отображаться количество счетов.
Счета клиентам, сумма
Данная статистика будет автоматически заполняться,ссылаясь на данные в Счетах клиентам. Здесь будет отображаться сумма счетов.
Заявки, количество
Данная статистика будет автоматически заполняться,ссылаясь на данные в Заявках (будет отображаться кол-во).
Заявки, сумма
Данная статистика будет автоматически заполняться,ссылаясь на данные в Заявках (будет отображаться сумма).
Период ввода
Выбирается период ввода статистики: “Дни”, “Недели”, “Месяц” “Год”. Обычно выбирают тип “Недели”.
Период просмотра
Период просмотра выбирается аналогичным образом. Так как финансовое планирование проводится 1 раз в неделю, то рекомендуется “Недели”.
Статистика +1
Если вводится статистика по подсчету количества звонков, встреч, обработанных заказов и т.д., тогда удобнее использовать этот “счётчик”. В Личном кабинете сотрудника он будет отображаться следующим образом:
Перевёрнутая
Выбираем этот параметр в том случае, когда график статистики должен стремится к Нулю. Т.е. уменьшение статистики – это хорошо, а увеличение – плохо. Например, производственный брак должен уменьшаться (т.е. стремиться к 0), число возвратов продукции должно стремиться к 0 и это хорошо и т.д.
Минимум и Максимум
Заданные параметры позволяют правильно задать масштаб графика статистики (чтобы зрительно он действительно показывал график, а не лежащую линию, например).
Статистика ЦКП (ценного конечного продукта)
Отмечается в том случае, если это – основная статистика должности, статистика продукта. (Например, если у менеджера по продажам есть три статистики: “звонки”, “встречи” и “сумма продаж”, то статистикой продукта будет “сумма продаж”. А “звонки” и “встречи” – статистики подпродуктов. Они лишь ведут к получению продукта.) У сотрудника может быть только одна статистика ЦКП.
Затем нажимаем кнопку “Далее”.
Третья страница “Дополнительно” – совет по заполнению статистики. В этом поле рекомендуется описать правила подсчета статистики.
Например: “В статистику считаются только полностью выполненные заказы. Если клиент еще не получил свой заказ, в статистику он не учитывается”. Или “Статистика считается следующим образом: 1 балл – нанятый стажер, 3 балла – переведенный в штат сотрудник”.
Затем нажмите “Далее”.
В случае, если создаётся Групповая, Мульти- или Формульная статистика, то на странице “Дополнительно” будут показаны другие статистики, которые можно включить в эту статистику в качестве вложенных. Их нужно отметить в списке “галочками” и сохранить.
В нашем случае, при создании обычной статистики, программа переходит к последнему пункту “Входит в статистики”. Если мы хотим вложить создаваемую статистику в групповую, формульную, или мульти-статистику, то отмечаем эти статистики галочкой и нажимаем “Создать”.
Созданная статистика отобразится следующим образом:
Строка вновь созданной статистики показывается на красном (светло-красном) фоне. Красный фон означает падающую статистику. Так как статистика только что создана, то в ней ещё нет данных. Статистика без данных считается падающей.
На этом процесс создания одной статистики завершается.
