# 2.08 Подключение Gmail-аккаунта электронной почты к ManaJet при помощи пароля приложения

Публичная ссылка: https://app.manajet.org/FAQ/Public/Index/322?culture=ru-RU

Не так давно Google сообщил, что для доступа к почте сторонних приложений будет необходимо включение двухфакторной аутентификации.
Если двухфакторная аутентификация не включена, то Gmail не предоставляет доступ к почте сторонним приложениям (таким, как ManaJet), даже если вводятся правильные логин и пароль. Тогда при добавлении/изменении аккаунта электронной почты в ManaJet появляется такое сообщение об ошибке:
Решение – включить двухфакторную аутентификацию и затем создать индивидуальный пароль приложения. Gmail считает доступ по такому паролю более безопасным, так как главный пароль пользователя при этом не раскрывается. Индивидуальный пароль пользователь может в любой момент удалить, и тогда внешнее приложение доступ к почте потеряет.
Нужно перейти в “Управление аккаунтом Google”.
Выбрать пункт “Безопасность”. (Если ваша почта находится в корпоративном домене, то этот пункт меню будет только у администратора домена. Т.е. авторизоваться необходимо через адрес, управляющий доменом.) Затем включить двухэтапную аутентификацию, выбрать подтверждение по смс, указать номер телефона и затем ввести полученный в смс код.
Теперь снова работаем с настройками подключаемого адреса (не администратора домена). Выбираем пункт “Пароли приложений”. (Кликом по надписи “Ни одного”).
Иногда пункт “Пароли приложений” отсутствует. В этих случаях нужно сделать поиск по тексту “Пароли приложения” наверху страницы:

Называем это приложение “ManaJet” и нажимаем “Создать”.

Копируем пароль в клипборд: (отмечаем все знаки пароля и нажимаем Ctrl/C в Windows или Cmd/C в MacOS).

Скопированный пароль вводим в поле “Пароль” в форме настройки аккаунта электронной почты в ManaJet. Затем нажимаем кнопку “Проверить”:
Для Gmail необходимы следующие настройки:
В “Исходящих” (отправка сообщений, подключение по протоколу SMTP):
Адрес сервера: smtp.gmail.com.
Включаем пункт “ССЛ” (подключение по защищённому протоколу)
Порт: 587.
Имя пользователя: ваш полный адрес в Gmail, включая @gmail.com – “moiadres@gmail.com”. Если на Gmail размещён ваш корпоративный домен, то полный адрес, включая имя этого домена:
Пароль: пароль приложения
Во “Входящих” (приём сообщений, протокол IMAP)
Адрес сервера: imap.gmail.com.
Включаем пункт “ССЛ” (подключение по защищённому протоколу)
Порт: 993.
Имя пользователя: ваш логин в Gmail полностью, включая @gmail.com (или включая имя корпоративного домена).
Пароль: пароль приложения.
