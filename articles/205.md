# 2.02.2.5 Создание автоматических статистик

Публичная ссылка: https://app.manajet.org/FAQ/Public/Index/205?culture=ru-RU

Автоматические статистики — это такой тип статистик, с помощью которых Ваша статистика заполняется автоматически, загружая данные из нужного раздела программы.
Типы автоматических статистик в нашей программе есть следующие:
Выручка — данные этой статистики берутся автоматически из раздела Финансового планирования. Здесь будут отображаться все доходы, поступившие в компанию за периоды финансового планирования в виде статистики.
Маржинальный доход — данные этой статистики берутся автоматически из раздела Финансового планирования. В эту статистику будут попадать данные по выручке, из которой уже вычтены отчисления в фонды выручки.
Скорректированный доход — данные этой статистики берутся автоматически из раздела Финансового планирования. В ней будет отражаться разница маржинального дохода и отчислений в фонды маржинального дохода. за периоды финансового планирования.
Расходы — данные этой статистики берутся автоматически из раздела Финансового планирования “Расходы”. В эту статистику попадают все расходы, которые были произведены компанией за периоды финансового планирования.
Одобренные расходы — данные этой статистики берутся автоматически из раздела Финансового планирования “Расходы”. В эту статистику попадают все одобренные расходы, которые были произведены компанией за периоды финансового планирования.
CRM: новые компании — в этой статистике будут отображаться данные, взятые из раздела “CRM” вкладки “Компании”.
Счета поставщиков, количество — данные этой статистики берутся автоматически из раздела Финансового планирования, вкладки “Счета поставщиков”. В статистике будет отображаться количество счетов поставщиков, которое было выставлено компании за периоды финансового планирования.
Счета поставщиков, сумма — данные этой статистики берутся автоматически из раздела Финансового планирования, вкладки “Счета поставщиков”. В статистике будет отображаться сумма счетов поставщиков, которая была выставлена компании за периоды финансового планирования.
Счета клиентам, количество — данные этой статистики берутся автоматически из раздела Финансового планирования, вкладки “Счета клиентам”. В статистике будет отображаться количество счетов клиентам, которое было выставлено компанией за периоды финансового планирования.
Счета клиентам, сумма — данные этой статистики берутся автоматически из раздела Финансового планирования, вкладки “Счета клиентам”. В статистике будет отображаться количество счетов клиентам, которое было выставлено компанией за периоды финансового планирования.
Заявки, количество — данные этой статистики берутся автоматически из раздела Финансового планирования, вкладки “Заявки”. В статистике будет отображаться количество заявок, созданное в компании за периоды финансового планирования.
Заявки, сумма — данные этой статистики берутся автоматически из раздела Финансового планирования, вкладки “Заявки”. В статистике будет отображаться сумма заявок, которая была запрошена в компании за периоды финансового планирования.
CRM: первые счета клиентам, сумма  — данные этой статистики берутся автоматически из раздела Финансового планирования, вкладки Счета клиентам. В статистике учитывается сумма выписанного (не оплаченного) счёта компании. В статистике отображаются данные нескольких компаний (или же частных лиц) сразу. , Таким образом можно понять сколько новых клиентов появилось и какую прибыль они Вам приносят.
CRM: первые счета клиентам, количество — данные этой статистики берутся автоматически из раздела Финансового планирования, вкладки Счета клиентам. В статистике учитывается количество выписанного (не оплаченного) счёта компании. В статистике отображаются данные нескольких компаний (или же частных лиц) сразу. Таким образом, можно отслеживать количество новых клиентов в Вашей компании за определённый промежуток времени.
Счета поставщиков. Сроки оплат, количество — данные этой статистики берутся автоматически из раздела Финансового планирования, вкладки счета поставщиков. С помощью планируемых счетов поставщиков можно отслеживать количество предположительных счетов. Это поможет предвидеть и запланировать бюджет компании.
Счета поставщиков. Сроки оплат, сумма — данные этой статистики берутся автоматически из раздела Финансового планирования, вкладки счета поставщиков. С помощью планируемых счетов поставщиков можно отслеживать сумму предположительных счетов. Это поможет предвидеть и запланировать бюджет компании.
Счета клиентам. Сроки оплат, количество — данные этой статистики берутся автоматически из раздела Финансового планирования, вкладки счета клиентам. С помощью планируемых счетов клиентам можно отслеживать количество предположительных счетов. Это поможет предвидеть и запланировать бюджет компании.
Счета клиентам. Сроки оплат, сумма — данные этой статистики берутся автоматически из раздела Финансового планирования, вкладки счета клиентам. С помощью планируемых счетов можно отслеживать сумму предположительных счетов клиентам. Это поможет предвидеть и запланировать бюджет компании.
Рассылка, отправлено — данные этой статистики берутся автоматически из раздела CRM, вкладки “Рассылка”. С помощью данной статистики можно отслеживать количество Вашего исходящего потока.
Рассылка, открыто — данные этой статистики берутся автоматически из раздела CRM, вкладки “Рассылка”. С помощью данной статистики можно отслеживать количество открытых получателями писем. Это поможет Вам определить, насколько привлекательны и примечательны ваши электронные письма, а также понять, какие темы или заголовки наиболее привлекают внимание вашей аудитории. На основе этой информации вы сможете улучшить свои электронные рассылки и увеличить эффективность вашей коммуникации.
Звонки — при синхронизации CallGear появляется отдельная вкладка  в CRM. Как раз-таки из этого раздела и берутся данные для данной статистики. С помощью данной статистики можно отслеживать общее количество проводимых сотрудником звонков.
Звонок, успешный — при синхронизации CallGear появляется отдельная вкладка  в CRM. Как раз-таки из этого раздела и берутся данные для данной статистики. С помощью данной статистики можно отслеживать количество успешных звонков, проводимых сотрудником.
Звонок, секунды — при синхронизации CallGear появляется отдельная вкладка  в CRM. Как раз-таки из этого раздела и берутся данные для данной статистики. С помощью данной статистики можно отслеживать количество секунд успешных звонков, проводимых сотрудником
Далее приступим к созданию статистики.
На первом шаге “Общие параметры” нужно написать название, выбрать ответственного (пост, за которым данная статистика будет закреплена), также при необходимости нужно выбрать из списка того, кто вводит (если статистику будет заполнять не тот человек, за которым она закреплена, а кто-то другой, допустим, подчиненный), далее нужно указать единицы измерения статистики (например: штуки, рубли) и нажать “далее”.

Период ввода
Выбирается период ввода статистики: “Дни”, “Недели”, “Месяц” “Год”. Обычно выбирают тип “Недели”.
Период просмотра
Период просмотра выбирается аналогичным образом. Так как финансовое планирование проводится 1 раз в неделю, то рекомендуется “Недели”.
Статистика “+1”
Если вводится статистика по подсчету количества звонков, встреч, обработанных заказов и т.д., тогда удобнее использовать этот “счётчик”. В Личном кабинете сотрудника он будет отображаться следующим образом:
Перевёрнутая
Выбираем этот параметр в том случае, когда график статистики должен стремится к Нулю. Т.е. уменьшение статистики – это хорошо, а увеличение – плохо. Например, производственный брак должен уменьшаться (т.е. стремиться к 0), число возвратов продукции должно стремиться к 0 и это хорошо и т.д.
Минимум и Максимум
Заданные параметры позволяют правильно задать масштаб графика статистики (чтобы зрительно он действительно показывал график, а не лежащую линию, например).
Переходим во вкладку “Дополнительно”:
Компания
Из выпадающего списка можно выбрать компанию, данные по которой будут отображаться в статистике.
Статистика ЦКП
Отмечается в том случае, если это – основная статистика должности, статистика продукта. (Например, если у менеджера по продажам есть три статистики: “звонки”, “встречи” и “сумма продаж”, то статистикой продукта будет “сумма продаж”. А “звонки” и “встречи” – статистики подпродуктов. Они лишь ведут к получению продукта.) У сотрудника может быть только одна статистика ЦКП.
Показывать в ИЦО
Если выбрать данный параметр, статистика будет отображаться в Структуре на должности ответственного за неё человека.
Статистика проектов
Данная статистика будет выводиться в разделе “Управление проектами”.
Превью в режиме сравнения
Данная статистика позволит просматривать данную статистику в режиме сравнения ( сравнение будет происходить между данными за один год с другими).
Показать внизу Структуры
Если выбрать эту отметку, статистика будет отображаться внизу раздела “Структура”, закрепившись за ответственным сотрудником.
Использовать в расчёте зарплаты
Данная отметка позволит статистике отображаться в разделе “Расчёт Зарплаты” блока финансового планирования.
Следующий шаг - “Входит в статистики”. Здесь можно выбрать, в какую статистику будет входить наша новая статистика.
Как только статистика настроена, нажимаем на кнопку  в правом нижнем углу.
