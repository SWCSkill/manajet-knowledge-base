## Направляющие формы (НФ) - то, что в справке называется «Бизнес-процессы»

**Внимание - терминологический мост.** В справке ManaJet это раздел 13 «Бизнес-процессы», и все интерфейсные надписи говорят «бизнес-процесс», «шаблон бизнес-процесса». В орг.политике SWC мы это называем **Направляющей формой (НФ, NF)** - термин из административной технологии HMS. По функциональности это одно и то же; по смыслу мы делаем дополнительный акцент: НФ это маршрутная карта движения частицы.

**Что такое частица.** Частица - всё, что требует обработки: документ, заказ, инвойс, накладная, деталь, новый сотрудник, клиент, заявка на партнёрство. НФ описывает её путь: какие посты к ней касаются, в какой последовательности, что делает каждый.

**Зачем НФ нужна.** Два ключевых результата:
- *Качество* - ни один шаг не теряется, потому что зашит в форме.
- *Скорость* - как в эстафете: каждый выполняет свой этап и передаёт следующему. Внутреннее давление одного сотрудника на другого создаётся без участия руководителя.

**Два способа выполнения.**
1. *Клиент выполняет сам* - физически ходит с формой от сотрудника к сотруднику. Действия в системе на клиента не назначаются.
2. *Сотрудники передают друг другу* - клиент форму не держит. Можно назначать системные действия: внести в CRM, создать счёт, проверить склад.

**Два формата.**
- *Бумажный* - форма распечатывается, данные вписываются ручкой. Для условий без компьютера или для гостевого сценария.
- *Электронный в ManaJet* - данные вносятся в программу, уведомления передаются автоматически, система фиксирует кто, когда и что выполнил. Быстрее и без переноса с бумаги.

### Создание шаблона НФ - порядок и нюансы

Полный технический how-to - в статьях 13.1, 13.2, 13.3 справки. Здесь - дополнительные SWC-рекомендации, которых в справке нет.

**Порядок работы:** сначала создай все шаги без детализации, потом настрой переходы между ними, потом распиши подшаги в каждом. Если делать иначе - запутаешься в связях.

**Шаги назначаются на подразделения, а не на конкретных сотрудников.** Уведомление приходит всем в подразделении, выполнит тот, кто первым нажмёт «Перенять». Так балансируется нагрузка и форма не висит, если один человек в отпуске.

**Регламент шага** - предельное время выполнения. Без регламента индикатор скорости не работает, и форма теряет одно из главных свойств - давление по времени.

**Два завершающих шага вместо одного.** Это рекомендация SWC, в справке её нет. Один шаг - нормальное завершение (форма выполнена как задумано). Второй - аварийный, когда форма не может быть завершена стандартно (отказ клиента, нерешаемая блокировка). На аварийном завершении - инспекция причины. Так мы не теряем статистику по сорванным процессам.

**К шагу можно прикрепить файл или ссылку на шляпный документ.** Ссылка предпочтительнее: при обновлении шляпы данные в форме обновляются автоматически.

### Подшаги и поля данных

Внутри каждого шага могут быть подшаги - конкретные действия с фиксацией результата. Типы полей: текст (одна строка), текст (много строк), десятичное число, дата, да/нет, загрузка файла, Компания (запись в CRM), счёт клиенту, счёт поставщику.

Две важные отметки:
- **«Обязательно для заполнения»** - без этого данного форма дальше не пойдёт.
- **«Показывать в списке»** - данное видно в списке форм и доступно для фильтрации. Используй для ключевых параметров (номер проекта, тип клиента и т.д.) - они потом понадобятся для аналитики и плазмы.

**Условный подшаг** - выполняется только при определённом условии. Используй, чтобы не плодить отдельные шаблоны на варианты одного процесса.

### Запуск и выполнение НФ

Запуск: блок НФ → «Начать NF» → выбрать шаблон → ввести данные первого шага → «Начать» (физическое действие, управление не передаётся) или «Сохранить и передать дальше» (шаг завершён, управление переходит следующему посту).

Выполнение шага: оранжевый флажок и всплывающее уведомление сообщают о новом шаге. Кликнуть → «Перенять» → ввести данные → выполнить подшаги → «Сохранить» (продолжаем работу) или «Сохранить и передать дальше».

Иконки в Личном кабинете:
- *Зелёная* - есть шаг в работе.
- *Красная с цифрой* - есть просроченные шаги. Это сигнал к немедленному действию.

### Индикатор скорости

В административной технологии: «Могущество определяется скоростью потока частиц». Индикатор скорости отображается в форме бабочки - цветная полоска.

- Середина (маленькая зелёная) - норма.
- Полоски влево (светло-зелёная → оранжевые → красные) - опоздание, чем дальше, тем больше.
- Полоски вправо (зелёные) - опережение нормы.

Красная зона - ситуация требует энергичных мер. Один задержанный шаг останавливает всю цепочку.

### Плазма для НФ

Каждый элемент НФ на плазме - шаг формы, обозначенный цветной угловой полоской. Форма выполняется слева направо. Цвет полоски - цвет отделения, выполняющего шаг.

Столбики внутри шага: кирпичный - входящие, синий - текущие, красный - просроченные, зелёный - завершённые. После полосок шагов идут итоговые столбики по всей форме (количество форм на линиях, не сумма шагов). После итоговых - спидометр: вертикально - норма, вправо ×2/×3 - опережение, влево - опоздание.

Добавить НФ на плазму: редактирование плазмы → добавить элемент (иконка НФ) → выбрать шаблон в закладке «Данные» → настроить период → сохранить.

### Отслеживание этапов проектов через НФ

Если у проекта несколько этапов - для каждого этапа отдельный шаблон НФ. В каждом шаблоне добавь шаг «Привязка к проекту» с полем типа «Десятичное число» и отметкой «Показывать в списке». При запуске формы вводится номер проекта.

Фильтрация: расширенный поиск → «Дополнительные колонки» → ввести номер проекта. Создать именованный фильтр (хранится бессрочно). Токен фильтра из адресной строки вставить в поле «Фильтр токен» компонента плазмы - плазма покажет только отфильтрованные формы по этому проекту.

## Принципы SWC, которые отражаются в работе с ManaJet

Эти принципы пронизывают использование инструмента, но в справке ManaJet их нет - они часть орг.политики SWC. Если сотрудник цитирует их или спрашивает - можешь подтвердить и дать пример применения, но за смыслом отправляй к `swc-assistant`.

**«Если действие не зафиксировано в ManaJet - управленчески оно не существует.»** Сделанная работа без записи (закрытая задача без подтверждения, выполненный шаг НФ без внесения данных, ввод статистики «потом») - не считается. ManaJet не журнал, а единственный регистратор фактов.

**«Могущество определяется скоростью потока частиц.»** Эффективность подразделения видна в том, как быстро через него проходят НФ. Не количество запущенных форм, а скорость их прохождения - ключевая метрика.

**«Послание адресуется посту, не человеку.»** В коммуникациях ManaJet всегда указывается отдел/отделение и пост. Если пост сменил исполнителя - история сохранится за должностью, а не уйдёт в личную почту ушедшего сотрудника.

**«Каждый пост имеет минимум одну числовую статистику, отражающую его производительность.»** Не активность - именно производительность. Статистика измеряет ЦКП поста.
